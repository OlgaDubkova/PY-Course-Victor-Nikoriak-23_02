# 🧩 TEAM 3 — health.py

## 📖 Що робить цей модуль

Цей модуль перевіряє чи гравець поїв сьогодні і змінює його **здоров'я**.

Якщо їжа є — гравець набирає сили. Якщо їжі немає — гравець слабшає від голоду.

---

## ⚙️ Що потрібно зробити

1. Перевірити значення `state["food"]`.
2. Якщо `food == 0`:
   - Відняти **10** від `state["health"]` (гравець голодує).
   - Надрукувати: `Health change: -10`
3. Якщо `food > 0`:
   - Додати **5** до `state["health"]` (гравець поїв).
   - Надрукувати: `Health change: +5`
4. Повернути оновлений `state`.

> **Увага:** `state["food"]` тільки читати — не змінювати його тут!

---

## 🧠 Які знання використовуються

- `if / else`
- `dict` — `state["food"]` і `state["health"]` — цілі числа

---

## 🔗 Як цей модуль впливає на інші

👉 Цей модуль залежить від **Team 1 (resources.py)**:
якщо Team 1 не реалізує додавання їжі → `food` завжди буде 0 → здоров'я падатиме щодня.

👉 Якщо `health` впаде до **0 або нижче** →
`main.py` виведе `"You died..."` і гра закінчиться.

👉 Кілька поганих днів без їжі → `-10` щодня → через 10 днів гравець мертвий.

## 📊 Приклади

**Приклад A — їжі немає:**

```python
state = {"health": 100, "energy": 90, "food": 0}
# food == 0 → health - 10
state["health"] = 90
print("Health change: -10")
```
```
Health change: -10
```

**Приклад B — їжа є:**

```python
state = {"health": 100, "energy": 90, "food": 7}
# food > 0 → health + 5
state["health"] = 105
print("Health change: +5")
```
```
Health change: +5
```

## ❗ Важливо

- Не змінювати інші модулі — тільки свій файл `health.py`
- Читати `state["food"]`, але **не змінювати** його — тільки `state["health"]`
- Обов'язково `return state` в кінці функції
- Назва функції залишається: `def run(state: dict) -> dict:`
