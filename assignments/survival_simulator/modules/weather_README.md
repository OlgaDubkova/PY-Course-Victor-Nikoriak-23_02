# 🧩 TEAM 2 — weather.py

## 📖 Що робить цей модуль

Цей модуль симулює **погоду дня** і застосовує її вплив на енергію гравця.

Кожен день погода обирається випадково: сонячно, дощ або шторм. Від погоди залежить скільки сил залишиться у гравця.

---

## ⚙️ Що потрібно зробити

1. Створити список можливої погоди: `["Sunny", "Rainy", "Storm"]`.
2. Випадково обрати одну з них за допомогою `random.choice()`.
3. Застосувати ефект:
   - `"Sunny"` → додати **+10** до `state["energy"]`
   - `"Rainy"` → нічого не змінювати
   - `"Storm"` → відняти **-15** від `state["energy"]`
4. Надрукувати яка погода сьогодні.
5. Повернути оновлений `state`.

---

## 🧠 Які знання використовуються

- `import random`
- `random.choice(list)` — обирає випадковий елемент зі списку
- `if / elif / else`
- `dict` — `state["energy"]` зберігає енергію гравця

---

## 🔗 Як цей модуль впливає на інші

👉 Якщо буде **Storm** кілька днів поспіль →
`energy` впаде до 0 → **GAME OVER** (гравець знесилений).

👉 **Sunny** дає +10 енергії → гравець краще переносить наступні події.

👉 **energy** перевіряється в `main.py` після кожного дня — якщо `<= 0`, гра зупиняється.

---

## 📊 Приклад

**Вхід:**
```python
state = {"health": 100, "energy": 100, "food": 0}
```

**Виконання** (random обрав "Storm"):
```python
weather = "Storm"
state["energy"] = 100 - 15  # = 85
print("Weather: Storm")
```

**Вихід:**
```
Weather: Storm
```
```python
state = {"health": 100, "energy": 85, "food": 0}
```

- Не змінювати інші модулі — тільки свій файл `weather.py`
